# 媒体转发

TeleBridge 完整支持各种媒体类型的转发，用户发送的媒体消息会转发给管理员。

## 支持的媒体类型

| 类型 | 说明 |
|------|------|
| 📷 图片 | 单张图片（Photo） |
| 🎥 视频 | 视频文件（Video） |
| 📄 文档 | PDF、Word、Excel 等（Document） |
| 🎵 音频 | 音乐文件（Audio） |
| 🎤 语音 | Telegram 语音消息（Voice） |
| 🎞 动画 | GIF 动画（Animation） |
| 🏷 贴纸 | Telegram 贴纸（Sticker） |

## 转发流程

### 纯媒体消息（无文字说明）

```
用户发送媒体 ──► 机器人转发原始媒体给管理员
                  │
                  └─► 发送用户信息（用户名、ID、时间）
```

转发时会先转发原始媒体消息，再发送一条包含用户信息的文本消息。

### 带文字说明的媒体（有 caption）

如果用户发送媒体时附带了文字说明（caption），机器人会：

1. **触发关键字匹配**：caption 文字会参与关键字匹配
   - 如匹配关键字，用户收到关键字回复
   - 如不匹配，用户收到默认自动回复（如已配置）
2. **转发原始媒体**：媒体和 caption 一起转发给管理员
3. **发送用户信息**：附带一条用户信息文本消息

::: tip caption 与关键字
媒体消息附带的文字说明（caption）和普通文字消息一样会触发关键字匹配。如不希望媒体触发关键字，可在 `bot.py` 中调整 `on_message` 函数的判断逻辑。
:::

## 管理员回复媒体

管理员收到转发的媒体消息后，可以直接**回复**该消息：

1. 点击转发的媒体消息
2. 选择「回复」
3. 输入回复内容（文字或媒体均可）
4. 机器人将回复内容发回给原始用户

机器人通过 `forward_tracker` 自动找到对应的用户，无需手动指定用户 ID。

## 注意事项

- 媒体大小受 Telegram API 限制（机器人下载最大 20MB，上传最大 50MB）
- 转发时保留原始消息的媒体信息（分辨率、时长、文件名等）
- 贴纸以原始格式转发（动画贴纸保持动画效果）
- 如转发失败，会在管理员聊天中显示错误信息

## 转发消息的格式

转发给管理员的消息格式：

**文字消息：**
```
👤 用户名 | @username | ID: 123456789 | 🕐 2026-07-18 12:34:56

💬 消息：
用户发送的消息内容
```

**媒体消息：**
```
[转发的原始媒体消息]

👤 用户名 | @username | ID: 123456789 | 🕐 2026-07-18 12:34:56
```

管理员可直接回复任意一条消息，机器人都会将回复发给对应用户。
