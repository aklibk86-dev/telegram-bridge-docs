# 自动回复

TeleBridge 支持灵活的自动回复机制，包括关键字匹配和默认回复。

## 关键字自动回复

在 `config.json` 或通过 `/settings` 面板配置关键字：

```json
{
  "keywords": {
    "价格": {
      "reply": "我们的基础套餐价格为 $99/月",
      "buttons": [{"text": "查看套餐", "url": "https://example.com/pricing"}]
    },
    "工作时间": {
      "reply": "我们的工作时间为周一至周五 9:00-18:00",
      "buttons": []
    }
  }
}
```

### 匹配规则

- 用户消息中**包含**关键字即触发（子字符串匹配）
- **不区分大小写**
- 匹配后发送关键字配置的回复内容
- 仅对**文字消息**或**带 caption 的媒体消息**生效（纯媒体不触发）

::: tip 关键字与转发的关系
关键字匹配**不会拦截**消息转发——匹配后用户收到关键字回复，消息**仍会**转发给管理员。这样管理员依然能看到所有用户消息，便于监控。

如需关键字匹配后不转发，需修改 `bot.py` 的 `on_message` 函数逻辑。
:::

## 默认自动回复

当用户消息**不匹配**任何关键字时，发送 `auto_reply` 字段配置的默认回复文本。

::: warning 注意
`auto_reply` 仅在**未匹配关键字**时发送。如已匹配关键字，不会重复发送默认自动回复。
:::

### 自动删除计时器

设置 `auto_reply_delete_time`（秒），自动回复消息会在指定时间后自动删除：

- 设置为 `10`：10 秒后删除
- 设置为 `0`：不删除（默认）
- 设置为空字符串或负数：视为 `0`

适用于临时通知场景，减少聊天界面干扰。

::: warning 删除权限
机器人只能删除自己发送的消息。删除失败（如权限不足）不会报错，消息会保留。
:::

## 完整处理流程

```
用户发送消息
  │
  ├─► 消息有文字（text 或 caption）？
  │     │
  │     ├─ 是 ─► 匹配关键字？
  │     │          │
  │     │          ├─ 匹配 ─► 发送关键字回复（带可选按钮）
  │     │          │
  │     │          └─ 不匹配 ─► 发送默认自动回复（如已配置）
  │     │
  │     └─ 否（纯媒体）──► 跳过关键字和自动回复
  │
  ▼
消息转发给管理员（始终执行）
```

::: tip 转发始终发生
无论是否匹配关键字，用户消息**都会**转发给管理员。这是 TeleBridge 的设计：管理员不应错过任何用户消息。
:::
