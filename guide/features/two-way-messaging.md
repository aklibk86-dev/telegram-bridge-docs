# 双向通讯

这是 TeleBridge 的核心功能，实现普通用户与管理员之间的双向消息中转。

## 工作流程

```
用户 ──► 发送消息 ──► 机器人 ──► 转发给管理员
                          │
                          ▼
用户 ◄── 接收回复 ◄── 机器人 ◄── 管理员回复
```

### 详细流程

1. **用户发送消息**：用户向机器人发送任意文本或媒体消息
2. **关键字匹配**（仅对文字消息或带 caption 的媒体）：如匹配关键字，用户收到关键字回复；如不匹配，用户收到默认自动回复（如已配置）
3. **转发管理员**：消息**始终**转发给管理员（无论是否匹配关键字），转发时附带用户信息和时间
4. **管理员回复**：管理员可以直接回复转发的消息，机器人通过 `forward_tracker` 找到原始用户
5. **消息送达**：将管理员的回复精准发送给原始用户

::: tip 关键字不拦截转发
关键字匹配**不会拦截**消息转发。这样设计是为了让管理员监控到所有用户消息。详见 [自动回复](./auto-reply)。
:::

## 快速回复机制

管理员在 Telegram 中收到转发的消息后，直接点击「回复」并输入内容，机器人会自动将回复发给对应的用户，无需手动指定用户 ID。

### 回复方式对比

| 方式 | 操作 | 适用场景 |
|------|------|----------|
| 直接回复 | 回复转发的消息 | 最常用，机器人自动识别用户 |
| `/reply` 命令 | `/reply <用户ID> <消息>` | 需要主动联系某用户时 |

### `/reply` 命令示例

```
/reply 123456789 你好，你的问题已处理。
```

用户 ID 可通过 `/users` 命令查看已注册用户列表获取。

## 管理员消息处理

管理员发送的消息（非回复转发消息）会触发管理员提示：

```
💡 管理员提示

• 回复转发消息 → 回复对应用户
• /settings → ⚙️ 交互式设置
• /broadcast <消息> → 广播所有用户
• /reply <ID> <消息> → 回复指定用户
• /stats → 查看统计
• /users → 查看用户列表
```

如管理员发送的内容匹配底部快捷键盘的按钮（如「⚙️ 设置」），会触发对应功能。

## 消息转发格式

转发给管理员的消息包含以下信息：

```
👤 用户名 | @username | ID: 123456789 | 🕐 2026-07-18 12:34:56

💬 消息：
用户发送的消息内容
```

- **用户名**：Telegram first name
- **@username**：用户名（如未设置则不显示）
- **ID**：用户唯一 ID（可用于 `/reply` 命令）
- **时间**：消息接收时间

## 使用场景

- **客服系统**：用户咨询问题，管理员在线解答
- **反馈收集**：用户提交反馈，管理员统一处理
- **匿名沟通**：保护用户隐私，管理员看不到用户的手机号等信息
- **通知广播**：管理员向所有用户群发重要通知
