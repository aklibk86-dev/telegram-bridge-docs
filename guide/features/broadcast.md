# 广播消息

管理员可以通过 `/broadcast` 命令向所有注册用户群发消息。

## 使用方式

### 基本用法

```
/broadcast 大家好，系统将于今晚 23:00-23:30 进行维护升级。
```

直接在 `/broadcast` 后输入要广播的内容即可，机器人会：

1. 显示「正在发送广播消息...」状态
2. 依次向所有注册用户发送消息
3. 完成后报告成功和失败数量

```
✅ 广播完成

📊 成功：15 人
❌ 失败：2 人
```

### 广播消息格式

用户收到的广播消息格式：

```
📢 广播通知

<你输入的广播内容>
```

如配置了 `broadcast_buttons`，广播消息会附带这些内联按钮。

## 配置广播按钮

广播消息附带的内联按钮通过 `config.json` 的 `broadcast_buttons` 字段配置：

```json
{
  "broadcast_buttons": [
    {"text": "查看详情", "url": "https://example.com"},
    {"text": "反馈问题", "url": "https://t.me/support"}
  ]
}
```

或在 `/settings` → 按钮管理 → 广播作用域 中配置。

::: tip 按钮预先配置
广播按钮需在广播前预先配置好，每次广播都会自动附带这些按钮。如需不同广播带不同按钮，需在每次广播前修改 `broadcast_buttons` 配置。
:::

## 注意事项

- 广播只会发送给**已注册**的用户（使用过 `/start` 的用户）
- 广播内容支持 HTML 格式（如 `<b>加粗</b>`、`<i>斜体</i>`）
- 大量用户时发送需要一些时间（受 Telegram API 频率限制，每条间隔约 50ms）
- 失败的用户通常是已封禁机器人或删除账号的用户
- 广播完成后会显示成功和失败的统计

## 示例

### 纯文字广播

```
/broadcast 🎉 新功能上线！现在支持关键字自动回复，详见 /help。
```

### 带 HTML 格式的广播

```
/broadcast 📢 <b>系统维护通知</b>

系统将于今晚 <i>23:00-23:30</i> 进行维护升级，期间可能短暂不可用。

感谢您的理解与支持！
```

### 配置按钮后广播

1. 先在 `/settings` → 按钮管理 → 广播 中添加按钮（如「查看详情」）
2. 再执行广播命令，消息会自动附带按钮

## 相关命令

| 命令 | 说明 |
|------|------|
| `/broadcast <消息>` | 向所有注册用户发送广播 |
| `/users` | 查看已注册用户列表（了解广播覆盖范围） |
| `/stats` | 查看用户总数 |
